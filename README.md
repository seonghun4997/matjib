# 맛집검수소 v2

카카오맵 데이터 + 네이버 재방문율을 교차 검증해, 검증된 맛집만 지도에 보여주는 서비스.

## 구조

```
고객      지도 + 유형 필터 + 동네 검색/요청 + 공유
  ↕
자동      매일 새벽 3시: 요청 동네 수집 → 오래된 동네 갱신 → 조작 의심 자동 숨김
  ↕
어드민    대시보드 · 수집 · 네이버 검증 · 가게 관리 · 설정
```

## 신뢰 2단계

| 배지 | 조건 |
|---|---|
| 무조건 맛집 보장 | 카카오 검증 + 최근 20개 리뷰 중 재방문 4개 이상 (수기 검증) |
| 맛집일 확률 높음 | 카카오 평점·리뷰·태그 기준 통과 (재방문 검증 전) |

## 분류
- 🍜 음식맛집 — '맛' 태그 비율이 기준 이상
- ✨ 분위기맛집 — '분위기' 태그 비율이 기준 이상
- 둘 다 넘으면 양쪽 모두에 노출

## 설치

1. **Supabase** — SQL Editor에서 `supabase/schema.sql` 실행 (이 파일 하나면 끝)
2. **GitHub** — 이 폴더 내용물 전부 업로드
3. **Vercel** — Import 후 환경변수 3개:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `NEXT_PUBLIC_ADMIN_PASS`

## 운영

- **수집**: /admin → 수집 탭 → 동네 입력 (여러 줄 가능) → 시작
- **네이버 검증**: /admin → 검증 탭 → [네이버 열기] → 최근 20개 리뷰 중 "N번째 방문" 개수 세기 → 입력
- **자동**: 매일 새벽 3시 자동 실행 (대시보드에서 수동 실행도 가능)
- **문제 발생 시**: /lab 에서 진단 실행 → 결과 공유

## 조작 감지

리뷰 작성자 프로필(리뷰 수·평균별점·팔로워)과 등록 시각을 분석해 0~100점:
- 만점 계정(평균 4.9+) 편중
- 신규·저활동 계정의 5점 리뷰
- 특정 2주에 리뷰 집중 등록
- 별점 획일화
- 태그 극단 편중

기준점(기본 60점) 이상은 자동 숨김. 어드민에서 근거 확인 후 되살릴 수 있음.
